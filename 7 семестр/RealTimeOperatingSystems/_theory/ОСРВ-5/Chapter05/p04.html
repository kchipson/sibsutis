5.4. Расширения QNX6 для сигналов

Дополнительные сигналы 57..64 предназначены исключительно для sigwaitinfo(). Они всегда заблокированы.

Механизм sigqueue / sigwaitinfo аналогичен механизму передачи импульса MsgSendPulse / MsgReceive, но не требует создания канала и, как следствие, не работает по сети.

MsgSend, прерываемый сигналом, может ввести сервер (MsgReceive) в затруднительное положение. Чтобы выйти из затруднения, сервер создает канал с помощью

ChannelCreate (_NTO_CHF_UNBLOCK);

Если нить, пославшая сообщение серверу и находящаяся в состоянии REPLY_BLK, получает сигнал, то в канал серверу посылается импульс, информирующий сервер об этой ситуации. Сервер должен дать какой-либо ответ клиенту, только после этого сигнал будет доставлен и разблокирует нить.
